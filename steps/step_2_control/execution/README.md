# Этап 2. Управление docker-контейнером — самостоятельная работа.

**Исходные данные:** исходный код фронтенд приложения в директории `frontend`.

**Цель:** создать docker-compose.yml, настроить сервис frontend и запустить его в Docker.
Дополнительно необходимо выполнить базовое обследование и мониторинг контейнеров.

**Задачи:**

1. Создайте файл `docker-compose.yml` в текущей директории.

2. Заполнить `docker-compose.yml`. В конфигурации должен быть описан сервис фронтенда со следующими требованиями:

   2.1. Используется образ фронтенда, собранный на предыдущем этапе.

   2.2. Задано имя контейнера: frontend-orders

   2.3. Указан корректный контекст сборки

   2.4. Открыт наружу порт 3001 (внутренний порт контейнера — 3000)

   2.5. Подключён файл переменных окружения (env_file)

3. Запустить контейнер в detached-режиме.

4. Провести мониторинг контейнера, посмотреть:

   4.1. логи контейнера `docker logs`

   4.2. активные процессы `docker top`

   4.3. потребление ресурсов `docker stats`

5. Провести обследование контейнера, с помощью `docker inspect <container_name>` определить:

   5.1. Переменные окружения, с которыми был запущен контейнер

   5.2. Стартовую команда контейнера (`CMD`)

   5.3. Образ, с которым был запущен контейнер

6. Выполнить вход в контейнер `docker exec -it <container> sh`:

   6.1. Посмотреть список файлов и директорий внутри контейнера.

# Этап 3. Монтирование — самостоятельная работа.

**Исходные данные:** исходный код бэкенд приложения в директории `backend`.

**Цель:** создать docker-compose.yml, настроить сервис backend, правильно смонтировать постоянные данные.

**Задачи:**

1. Создайте файл `docker-compose.yml` в директории `execution/`.

2. Заполнить `docker-compose.yml`. В конфигурации должен быть описан **сервис бэкенда** со следующими требованиями:

   2.1. Используется образ бэкэнда, собранный на предыдущем этапе.

   2.2. Задано имя контейнера: `c-backend-orders-api`

   2.3. Указан корректный контекст сборки

   2.4. Открыт наружу порт 8081 (внутренний порт контейнера — 8080)

   2.5. Примонтирован volume для сохранности БД `/data/orders.db`

3. Объявить volume в `docker-compose.yml`.

4. Запустить контейнер.

5. Провести обследование контейнера, с помощью `docker inspect <container_name>` определить:

   5.1. Местоположение volume’а в системе

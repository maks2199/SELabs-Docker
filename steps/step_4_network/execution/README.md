# Этап 4. Сети — самостоятельная работа.

**Исходные данные:** исходный код бэкенд и фронтенд приложений в директориях `backend` и `frontend`.

**Цель:** создать docker-compose.yml, настроить сервисы backend и frontend, настроить их сетевое взаимодействие.

**Задачи:**

1. Создайть файл `docker-compose.yml` в текущей директории.

2. Заполнить `docker-compose.yml`. В конфигурации должны быть описаны сервисы бэкенда и фронтенда со следующими требованиями:

   **frontend:**

   2.1. Используется образ фронтенда, собранный на предыдущем этапе.

   2.2. Задано имя контейнера: frontend-orders

   2.3. Указан корректный контекст сборки

   2.4. Открыт наружу порт 3001 (внутренний порт контейнера — 3000)

   2.5. Указана сеть, общая с бэкендом

   **backend:**

   2.1. Используется образ бэкэнда, собранный на предыдущем этапе.

   2.2. Задано имя контейнера: `backend-orders-api`

   2.3. Указан корректный контекст сборки

   2.4. Указана сеть, общая с фронтендом

3. Объявить сеть в `docker-compose.yml`.

4. Запустить контейнер.

5. Проверить доступность контейнеров

   5.1. Фронтенд доступен из браузера по адресу <ip-вашей-ВМ>:3001

   5.2. Бэкенд НЕ доступен из браузера по адресу <ip-вашей-ВМ>:8081

   5.3. Фронтенду доступен бэкенд по внутренней docker-сети: отображается список заказов.

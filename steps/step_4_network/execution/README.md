# Этап 4. Сети и тома — самостоятельная работа.

**Исходные данные:** исходный код бэкенд и фронтенд приложений в директориях `backend` и `frontend`.

**Цель:** создать docker-compose.yml, настроить сервисы backend и frontend, настроить их сетевое взаимодействие.

**Задачи:**

1. Создайть файл `docker-compose.yml` в директории `execution/`.

2. Заполнить `docker-compose.yml`. В конфигурации должны быть описаны **сервисы бэкенда и фронтенда** со следующими требованиями:

   **frontend:**

   2.1.1 Используется образ фронтенда, собранный на предыдущем этапе.

   2.1.2. Задано имя контейнера: `c-frontend-orders`

   2.1.3. Указан корректный контекст сборки

   2.1.4. Открыт наружу порт 3001 (внутренний порт контейнера — 3000)

   2.1.5. Указана сеть, общая с бэкендом

   2.1.6. Указана переменная среды, dns-имя бэкэнда environment: BACKEND_HOST: orders-api

   **backend:**

   2.2.1. Используется образ бэкэнда, собранный на предыдущем этапе.

   2.2.2. Задано имя контейнера: `c-backend-orders-api`

   2.2.3. Указан корректный контекст сборки

   2.2.4. Указана сеть, общая с фронтендом

   2.2.5. Примонтирован volume для сохранности БД `/data/orders.db`

3. Объявить сеть в `docker-compose.yml`.

4. Объявить volume в `docker-compose.yml`.

5. Запустить контейнер.

6. Проверить доступность контейнеров

   6.1. Фронтенд доступен из браузера по адресу <ip-вашей-ВМ>:3001

   6.2. Бэкенд НЕ доступен из браузера по адресу <ip-вашей-ВМ>:8081

   6.3. Фронтенду доступен бэкенд по внутренней docker-сети: отображается список заказов.

   ![](./screenshot_19102025_111553.jpg)

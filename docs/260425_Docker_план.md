## Подготовка слушателей к работе

### 1. Настроить VSCode

- Установить VSCode: https://code.visualstudio.com/download
- Установить расширение Remote SSH: https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-ssh

### 2. Настроить VPN

- Установить OpenVPN Connect (рекомендуется): https://openvpn.net/client/ → Downloads → Windows → OpenVPN Connect
- Открыть OpenVPN Connect → File → Import → From local file → выбрать `.ovpn` файл, предоставленный администратором
- Подключиться: нажать Connect → ввести логин/пароль (если требуется)
- Разрешить доступ в брандмауэре Windows для OpenVPN (галочки для Private и Public сетей)

### 3. Подключиться к ВМ

Подробная инструкция: https://code.visualstudio.com/docs/remote/ssh

1. В левом нижнем углу VSCode нажать на две стрелочки **Open a Remote Window**
2. Выбрать **Connect to Host...** → **Configure SSH Host...** → выбрать файл `config`  
   _(Если файла `config` не существует — создать его)_
3. Добавить в файл конфигурацию для подключения к учебной ВМ:

```
Host se_labs_user
  HostName 192.168.2.1XX  # XX - ваш номер хоста
  User linux
  StrictHostKeyChecking no
  UserKnownHostsFile /dev/null
```

4. Выбрать **Connect to Host...** → **Configure SSH Host...** → выбрать `se_labs_user`
5. Ввести пароль `linux`

#### * Диагностика неисправностей (под вопросом)

**Security & Trust Issues**

- Image may contain malware, backdoors, or crypto miners if from an unverified source
- It could be an abandoned image with unpatched CVEs
- Image might embed secrets (tokens, SSH keys)

```bash
docker scout cves <image>
```

**Compatibility Problems**

- Wrong architecture (ARM vs AMD64)
- Requires kernel features your host doesn't support
- Depends on external services (DB, API) that aren't available

```bash
docker inspect --format='{{.Architecture}}' <image>
```

**Hidden Misconfigurations**

- Needs specific environment variables
- Expects mounted volumes or files
- Uses hardcoded ports that clash with your host

```bash
docker inspect <image>  # Check Env, Volumes, ExposedPorts sections
```

**Чеклист диагностики скачанного образа:**

- [ ] `docker inspect <image>` → посмотреть `Entrypoint`, `Cmd`, `Env`, `Volumes`, `Ports`
- [ ] `docker history <image>` → проверить на неожиданные слои
- [ ] Запустить security scan (Trivy / Docker Scout)
- [ ] Запустить контейнер интерактивно (`--entrypoint sh`) для ручного тестирования
- [ ] Проверить логи: `docker logs`
- [ ] При ошибке — запустить с `--rm -it` и отлаживать внутри

### 3. Монтирование — 30 мин

#### Введение (5 мин)

- Что такое маунты и зачем они нужны
- Основные сценарии использования: хранение данных между перезапусками, совместная работа приложения и хоста, шаринг файлов

#### Маунты на примере простого веб-приложения (10 мин)

1. Указать IP своей ВМ в `example/mount/public/config.json`

2. Запуск контейнера **без** маунта:

```bash
make example-no-mount
```

   Добавить файлы в браузере → перезапустить контейнер → убедиться, что данные стерлись.

```bash
make example-down
```

3. Запуск контейнера **с** маунтом:  
   Показать в `docker-compose.yml`:
